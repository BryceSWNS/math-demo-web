# 题目上传规范（老师参考）

本文用于指导老师在本项目中正确发布数学题目，避免格式错误导致展示异常。

## 适用页面

- 发布页：`/problems/new`
- 题目详情展示页：`/problems/[id]`

---

## 字段填写规则

### 1) 标题（必填）

- 类型：纯文本
- 长度：`3 ~ 120` 字符
- 建议：简洁描述题目主题，例如“二次函数与面积最值”

### 2) 身份与昵称（必填）

- 身份：老师 / 学生（二选一）
- 昵称：`2 ~ 24` 字符
- 说明：当前无登录，昵称用于展示发布者

### 3) 题干 `stemMd`（必填）

- 支持：`Markdown + LaTeX`
- 最少：1 个字符
- 推荐写法：
  - 行内公式：`设函数 $f(x)=x^2+1$`
  - 块公式：
    ```md
    $$
    \int_0^1 x^2\,dx=\frac{1}{3}
    $$
    ```
- 排版建议：若希望题干更紧凑，优先使用行内公式 `$...$`，仅在需要单独居中展示时再用 `$$...$$`

### 4) 选项 `optionsRaw`（可选）

- 按行填写，每行一个选项
- 推荐格式：
  ```text
  A. 1/3
  B. 1/2
  C. 1
  D. 2
  ```
- 规则说明：
  - 若使用 `A.` / `B.` / `C.` 前缀，会按前缀作为选项键
  - 若不写前缀，系统会自动按 `A/B/C/...` 编号
  - 选项内容支持 LaTeX；推荐写法：`A. 根为 $2$ 和 $3$，且 $x_1^2+x_2^2=13$`
  - 系统会尝试自动识别如 `x1^2+x2^2=13` 这类常见表达式并渲染，但复杂公式仍建议显式使用 `$...$`

### 5) 答案 `answerMd`（可选）

- 支持：`Markdown + LaTeX`
- 用于给出最终结论或关键步骤

### 6) 解析 `analysisMd`（可选）

- 支持：`Markdown + LaTeX`
- 建议分步骤说明推导过程，便于学生阅读

### 7) 标签 `tagsRaw`（可选）

- 用英文逗号分隔
- 示例：`函数,极限,微积分`
- 注意：不要使用中文逗号 `，`

### 8) 难度 `difficulty`（必选）

- 可选值：`easy` / `medium` / `hard`
- 页面上显示为：简单 / 中等 / 困难

### 9) 附件（可选）

- 支持类型：图片、PDF
- 支持多文件上传
- 建议：文件名避免特殊符号，便于后续管理

---

## Markdown 与 LaTeX 速查

### 常用 Markdown

- 一级标题：`# 标题`
- 列表：`- 项目1`
- 加粗：`**重点**`
- 代码块：
  ````
  ```text
  示例
  ```
  ````

### 常用 LaTeX

- 分数：`$\frac{a}{b}$`
- 根号：`$\sqrt{x}$`
- 上下标：`$x^2,\ a_n$`
- 求和：`$\sum_{i=1}^{n} i$`
- 积分：`$\int_0^1 x^2\,dx$`

---

## 常见问题与排查

### 1) 公式没有正常渲染

- 检查 `$...$` 或 `$$...$$` 是否成对闭合
- 检查是否误用了中文符号（如全角括号）
- 不要把公式分隔符写成 `\$` 或 `\$\$`（这是“转义后文本”，不是正常公式分隔符）
- 系统已兼容历史数据中的 `\$` 写法，但新题目仍建议直接使用 `$` / `$$`

### 2) 选项显示不对

- 确认每个选项单独占一行
- 建议统一使用 `A. ...` 格式，避免歧义

### 3) 标签没有分开

- 检查是否使用了英文逗号 `,`

### 4) 上传后页面报错

- 先确认必填项是否填写完整（标题、题干、昵称等）
- 再检查附件类型是否为图片/PDF

## 可直接复制的题目模板

```md
标题：二次函数练习题

题干：
已知函数
$$
y=-x^2+4x+5
$$
求其与 $x$ 轴交点坐标。

选项：
A. $(-1,0)$ 与 $(5,0)$
B. $(1,0)$ 与 $(5,0)$
C. $(-1,0)$ 与 $(3,0)$

答案：
A

解析：
令 $y=0$，得
$$
-x^2+4x+5=0
$$
化简得
$$
x^2-4x-5=0=(x-5)(x+1)
$$
故交点为 $(-1,0)$ 与 $(5,0)$。
```
